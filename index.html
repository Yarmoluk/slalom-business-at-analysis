What is business@?

Slalom's internal AI chat platform ("Slalom Chat") providing employees access to 30+ LLMs from 6 providers through a unified interface. Built on top of OpenAI, Amazon Bedrock, and Google Gemini APIs. Features include Chat, Chat History, Prompt Library, Digital Experts, and Model Advisor.

Version: ui v1.1.308 | api v1.0.455 (still in beta)

Platform Components

  • Chat - Direct model interaction with file upload
  • Model Advisor - Static catalog of 30+ models with filtering
  • Prompt Library - Pre-built prompts (read-only; submissions "later this summer")
  • Digital Experts - 6 purpose-built AI agents on GPT-4o/4o-mini
  • Chat History - Conversation persistence

Deficiencies & Gaps - For Eric

Eric: 10 Critical Observations

Gap 1 All Digital Experts Run on GPT-4o/4o-mini - Why Not Frontier Models?
Slalom gives employees access to Opus 4.5 and GPT-5.2 for chat, but every Digital Expert (RFP Analyzer, Customer Story Generator, Summit Advisor) runs on GPT-4o or 4o-mini - models that are now 2 generations behind. The RFP Analyzer, arguably the highest-value tool on the platform, is using a mid-tier model from 2024. If these experts used GPT-5 or Claude Sonnet 4.5, the output quality would jump significantly. This is a cost optimization decision that sacrifices quality where it matters most.
Gap 2 Only 6 Digital Experts for 11,000 Employees
Six agents for an entire $2.5B consulting firm. Compare: Connexus Advisor (meta-routing), Customer Story Generator (sales), Digital Expert Architect (meta-creation), RFP Analyzer (sales), Brand Reviewer (marketing), Summit Advisor (delivery). Missing entirely: technical architecture advisor, data engineering assistant, cloud migration helper, staffing/resource optimizer, SOW generator, competitive intelligence, interview prep, onboarding buddy. The platform has the infrastructure for agents but hasn't invested in building them out.
Gap 3 Prompt Library is Read-Only (Employee Contributions "Later This Summer")
The Prompt Library explicitly states employees can only "view and use existing prompts" and that contributions start "later this summer." For a firm that advocates for AI democratization, locking 11,000 consultants out of contributing their prompts is a bottleneck. The most valuable prompts come from practitioners, not a central team. Every day this stays locked, institutional knowledge stays trapped in individual chats.
Gap 4 No Agentic Framework - Chat Is 2024, Agents Are 2026
Slalom's own research shows only 39% of companies have achieved multi-agent workflows. The business@ platform has zero visible agentic orchestration - no multi-step pipelines, no model chaining, no MCP (Model Context Protocol), no tool-use beyond basic file upload. Competitors (Accenture, Deloitte, McKinsey) are building agent platforms. Slalom is still in the "chat with a model" phase. The Digital Expert Architect hints at the right direction but stops at system prompt generation.
Gap 5 No Governance, Usage Analytics, or Data Classification
No visible: usage dashboards (which models do employees actually use?), cost tracking per team/model, PII detection before submission, data classification tiers (which models can see client data?), or audit trails. DeepSeek-R1 routes to Chinese-hosted infrastructure - can someone paste client data into it? For a firm handling Fortune 500 client data, this is a compliance risk. Slalom's own research flags that only 42% of companies have model controls - they appear to be in the other 58%.
Gap 6 No RAG / Knowledge Layer
30+ models, all running on public training data. No indication of a Slalom-specific retrieval layer indexing internal methodologies, past proposals, delivery frameworks, or client engagement patterns. The Summit Advisor likely has a system prompt with methodology content, but that's a prompt, not a scalable knowledge architecture. A RAG layer would make every model "Slalom-aware" instantly.
Gap 7 Anthropic Limited to 3 Languages (vs. OpenAI's 27)
Every Claude model lists only English, Spanish, Japanese. OpenAI supports 27 languages. Google supports 13. For a global firm with offices in Latin America, Europe, and Asia-Pacific, this creates a two-tier employee experience. Non-English consultants are funneled to OpenAI regardless of task fit. Either the platform needs language-based auto-routing, or the language metadata is wrong and needs correction.
Gap 8 No Image Generation or Multimodal Output
The platform handles input multimodality (file upload, image analysis) but has no visible image generation (DALL-E, Imagen, Stable Diffusion, Ideogram). Consultants building decks, wireframes, diagrams, or visual deliverables must leave the platform. This fragments workflow and pushes employees to unsanctioned tools.
Gap 9 Missing Providers: No Mistral, Cohere, or xAI
Mistral (EU data sovereignty, GDPR-compliant inference), Cohere (best-in-class enterprise RAG with embed + rerank), and xAI Grok (real-time data access) are all absent. For European client engagements requiring data residency, there's no sovereign AI option. For building enterprise search solutions, Slalom consultants can't prototype with the leading RAG platform.
Gap 10 Model Advisor is a Static Catalog, Not an Intelligent Router
30+ models with subtle differences - overwhelming for non-technical consultants. The Model Advisor appears to be a filterable list (by provider, tool, file type) rather than an intelligent router that auto-selects based on task type, input language, file size, or cost tier. Risk: employees default to the model they already know (probably GPT-4o) and never discover that Gemini 3.0 can handle their 40MB PDF or that Opus 4.5 would be better for their complex architecture question.
